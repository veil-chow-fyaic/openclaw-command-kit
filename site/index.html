Scope safety

No actor, no route, no action.

Session history is sensitive. Command Kit resolves actor scope and route scope before listing, filtering, or restoring anything.

Actor

No actor, no action.

senderId provider accountId organization

Route

No route, no action.

provider accountId organization chatType sessionKey
  • Search happens only after scoped filtering.
  • Display labels are never authorization.
  • Normal responses never expose raw session IDs.

Restore path

The selected generation moves only after validation.

/resume N recomputes the scoped list, maps the displayed index, validates ownership, restores with backup, and reports success only after OpenClaw read-back.

Install from source

The supported release path is a source checkout linked into OpenClaw.

  1. Clone the repository.
  2. Install dependencies and build all packages.
  3. Symlink packages/plugin into OpenClaw extensions.
  4. Configure OpenClaw and restart the gateway.
  5. Verify with /sessions.

npm package names are reserved for a future manual release. They are not the current supported install path.

Architecture

OpenClaw remains the source of truth for sessions.

Channel Message OpenClaw Core Dispatch Extension Plugin registerCommand() Scope Deriver Core Command Router SessionHistoryService / RestoreService ResponseFormatter Same Channel Reply

packages/core

Command router, scope resolvers, session history service, restore service, response formatter, and gateway client.

packages/plugin

OpenClaw plugin entry, command handlers, scope reverse-lookup, and plugin manifest.

  • No WeCom Side Panel fallback.
  • No local user/contact/session mapping pool.
  • No generic agent-runtime abstraction.
